Situation & Task

Maintaining a stable aquatic environment in a fish tank requires constant monitoring and manual intervention for tasks like feeding, temperature regulation, and water changes. The objective of this project was to design and build a cost-effective, automated "Smart Fish Tank" that could perform these tasks autonomously and provide remote monitoring capabilities to the user via the Internet of Things (IoT).

My Role & Responsibilities:

As the project lead, I was responsible for the entire development process. This included selecting all electronic components, designing the complete circuit, programming the ESP8266 microcontroller in the Arduino IDE, developing the IoT dashboard on the Blynk platform, and assembling and testing the physical prototype.

Action

I developed a multi-faceted system by integrating various hardware and software components:

1. Hardware Integration

I selected and interfaced a suite of sensors with the ESP8266 NodeMCU, including a DS18B20 waterproof temperature sensor, a water level sensor, a PIR sensor for fish motion, and an ultrasonic sensor to monitor the food level in the automated feeder.

2. Actuator Control

I implemented control logic for multiple actuators: an L298N motor driver to power air and water pumps, a servo motor for the automated feeder mechanism, and a relay module to control the lighting system based on a real-time clock (RTC).

3. IoT Dashboard Development

Using the built-in Wi-Fi capabilities of the ESP8266, I connected the system to the Blynk cloud platform. I designed a custom dashboard, accessible via web and mobile, to display real-time data (temperature, water level, etc.) and the status of all pumps and feeders.

Result

The project successfully resulted in a fully functional prototype that automated all core aspects of aquarium maintenance and provided robust remote monitoring.

Full Automation Achieved

The system successfully controlled water temperature within the optimal 23-27°C range, automatically topped up the water level, and dispensed food on a schedule (6 am and 5 pm).

Remote Monitoring & Alerts

The Blynk dashboard provided real-time visibility of all system parameters. The system was also programmed to send notifications and emails if no fish motion was detected, alerting the user to potential issues.

Key Learnings

  • This project provided extensive hands-on experience in interfacing a variety of sensors and actuators with a microcontroller, including managing different signal types (analog, digital, I2C, 1-Wire).
  • I gained practical skills in IoT development, from establishing reliable Wi-Fi communication to designing a user-friendly cloud dashboard for data visualization and system control.
  • I learned to overcome real-world hardware challenges, such as waterproofing sensors for aquatic use and implementing a voltage-dropping circuit with diodes to safely power a lower-voltage air pump.
